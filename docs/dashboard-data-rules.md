# Dashboard Data Rules

## 목적

대시보드 집계는 다운로드된 체류외국인 raw 엑셀과 요청서 [IB 입국객 대시보드 요청_260325.xlsx](/C:/Codex/kresident/Plan/Dashboard/IB%20입국객%20대시보드%20요청_260325.xlsx)를 연결하기 위한 기준 문서다.

## 입력 소스

- 원본 파일 인덱스: [`data/metadata/download-registry.json`](/C:/Codex/kresident/data/metadata/download-registry.json)
- 원본 엑셀 저장 루트: [`data/raw`](/C:/Codex/kresident/data/raw)

집계 파이프라인은 registry를 기준으로 처리 대상 파일을 확정한다. 브라우저는 raw xlsx를 직접 읽지 않고, 사전 집계된 dataset만 사용한다.

## 단기 관광객 정의

요청서 기준 단기 관광객 수는 아래 비자 코드의 합계다.

- `B1(사증면제)`
- `B2(관광통과)`
- `C1(일시취재)`
- `C3(단기방문)`
- `C4(단기취업)`

대시보드의 모든 핵심 지표는 위 5개 컬럼 합산값을 기준으로 계산한다.

## 확인된 raw 파일 스키마 차이

### 2024-2026 계열

- 시트명 예: `시트_1_crosstab`
- 헤더 행: 보통 3행
- 주요 컬럼: `대륙`, `국적`, `성별`, `총합계`, `B1(사증면제)` ... `기타`
- 성별 값: `총계`, `남성`, `여성`

### 2015-2018 계열

- 시트명 예: `Sheet1`
- 헤더 행: 보통 2행
- 주요 컬럼: `국적∙지역` 또는 `국적명`, `총계`, `성별`, `소계`, `사증면제(B-1)` ... `기타`
- 성별 값: `(T)`, `(M)`, `(F)`
- 일부 파일은 본문 메타가 깨져 있어 국가명 공백이 불규칙하다.

## 표준화 규칙

### 월 식별

- 1차 기준: registry의 `articleTitle`에서 `YYYY년 M월`
- 2차 기준: 원본 시트 제목의 `(... YYYY. M. DD.현재)` 패턴
- 대시보드 period key: `YYYY-MM`

### 성별 매핑

- `총계`, `(T)` -> `total`
- `남성`, `(M)` -> `male`
- `여성`, `(F)` -> `female`

### 국가명 정리

- 앞뒤 공백 제거
- 중간 다중 공백을 단일 공백으로 정규화
- 예: `중      국` -> `중 국`처럼 단순 압축만 하면 오히려 부정확할 수 있으므로, 초기에 원문 보존 + trimmed 버전을 함께 검토한다
- 원문 표시는 보존하되 국가군 매칭은 canonicalization 비교를 사용한다
- canonicalization은 공백 제거와 일부 구두점 제거를 포함해 `타 이 완`, `캐 나 다`, `몽 골`, `프 랑 스` 같은 표기 차이를 흡수한다
- 최종 표준화는 구현 단계에서 별도 매핑 테이블과 canonical alias index로 보완한다

### 대륙 컬럼 해석

- `2019-12` 이후 계열은 원본에 `대륙` 컬럼이 존재한다
- `2019-11` 이전 계열은 대륙 컬럼이 없을 수 있으며 이 경우 상세 row의 `continentName`은 `null`이 될 수 있다
- `기타` 국가군은 특정 대륙 버킷이 아니라 명시 국가군 외 모든 국가의 합산이므로 단일 `continentName`을 부여하지 않는다

### 집계 대상 행

- 국가별 상세 행을 기본 단위로 사용
- 합계/총계 행은 국가별 상세 집계와 분리해 trend 및 share 계산의 기준값으로만 사용
- `기타` 국가군은 명시된 18개 국가군 외 모든 국가의 합산이며, 대륙 구분은 비워 둔다

## 대시보드 출력 구조

### 월별 트렌드

- 단위: 월별 전체 단기 관광객 수
- 데이터 소스: 월별 `total` 행 합계
- 시각화: line chart

### 상위 10개국 비중

- 단위: 선택 기준 시점의 국가별 단기 관광객 수와 비율
- 데이터 소스: 국가별 `total` 행
- 기본값 제안: 최신 월 기준 top 10
- 시각화: horizontal bar chart

### 성별 비중

- 단위: 선택 시점의 `male`, `female` 단기 관광객 수와 비율
- 데이터 소스: 월 합계 기준 성별 행
- 시각화: donut chart 또는 stacked bar

### 상세 표

- 기본 컬럼 제안:
  - `연도`
  - `월`
  - `국가`
  - `대륙`
  - `단기 관광객 수`
  - `남성 단기 관광객 수`
  - `여성 단기 관광객 수`
  - `월 전체 대비 비율`

## GitHub Pages 제약 반영

- 최종 대시보드는 `site/` 정적 파일로 배포한다.
- 집계 결과는 `site/data/dashboard_data.json` 같은 정적 산출물로 만든다.
- 상세 표 다운로드는 브라우저의 client-side export로 처리한다.

## 구현 단계에서 주의할 점

- 연도별 스키마 차이로 인해 헤더 탐색 로직이 필요하다.
- 일부 국가명은 공백/표기 방식이 불규칙하므로 표준화 테이블 확장성을 남겨야 한다.
- raw xlsx 개수가 많아졌으므로, 집계는 브라우저가 아니라 사전 생성 파이프라인에서 수행해야 한다.
